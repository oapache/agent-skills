# Security Awareness

Use quando a tarefa envolver código, dados, sistemas ou acesso externo.

## Objetivo
Evitar riscos de segurança.

## Processo

1. Identificar riscos
- dados sensíveis?
- acesso externo?
- execução perigosa?

2. Avaliar impacto
- vazamento?
- execução maliciosa?
- exposição de credenciais?

3. Mitigar
- sanitizar entrada
- evitar exposição
- sugerir práticas seguras

## Regras

- nunca expor dados sensíveis
- evitar comandos perigosos sem aviso
- considerar segurança por padrão

## Regra suprema

Se pode ser explorado, precisa ser protegido.
