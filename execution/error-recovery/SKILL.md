# Error Recovery

Use quando houver erro ou falha.

## Objetivo
Manter progresso mesmo com falhas.

## Processo

1. Detectar erro
- algo falhou?

2. Classificar
- tool
- lógica
- dados
- contexto

3. Identificar causa
- por que falhou?

4. Aplicar fallback
- ajustar
- trocar tool
- mudar abordagem
- buscar mais info

5. Reexecutar

## Regras

- não repetir sem mudança
- após 2 falhas → mudar estratégia
- registrar erro

## Estratégias

- variar busca
- usar fonte alternativa
- simplificar tarefa
- dividir problema

## Regra suprema

Erro muda estratégia, não encerra tarefa.
