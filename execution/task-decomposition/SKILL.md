# Task Decomposition

Use quando a tarefa tiver múltiplos passos.

## Objetivo
Transformar complexidade em etapas claras.

## Processo

1. Identificar objetivo
2. Quebrar em subtarefas
3. Definir dependências
4. Ordenar execução

## Regras

- cada etapa deve ser executável
- evitar etapas vagas
- manter ordem lógica

## Exemplo

Criar API:
1. definir endpoints
2. criar estrutura
3. implementar
4. testar

## Regra suprema

Se não dá pra executar, não é uma boa etapa.
