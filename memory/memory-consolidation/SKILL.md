# Memory Consolidation

Use esta skill ao final de tarefas relevantes, no fechamento do dia, ou sempre que houver acúmulo de contexto operacional que precise ser transformado em memória durável.

## Objetivo

Garantir que a memória persistente mantenha apenas conhecimento útil, estável e reutilizável, evitando que MEMORY.md vire um diário bagunçado.

## Conceito central

Nem tudo que aconteceu merece virar memória durável.

O agente deve distinguir entre:
- evento temporário
- progresso operacional
- conhecimento estável
- preferência recorrente
- decisão de projeto
- procedimento comprovado

## Fontes de consolidação

Consolidar a partir de:
- memory/YYYY-MM-DD.md
- resultados de tarefas concluídas
- erros recorrentes
- decisões repetidas
- preferências claramente consistentes
- workflows que funcionaram bem mais de uma vez

## Destinos de consolidação

### Promover para MEMORY.md
Quando a informação for:
- durável
- reutilizável
- importante
- recorrente
- válida além da tarefa atual

### Manter apenas no log diário
Quando a informação for:
- temporária
- específica de um único momento
- já superada
- irrelevante para sessões futuras

## O que promover

### Preferências estáveis
- estilo de resposta preferido
- ferramentas preferidas
- stack preferida
- restrições recorrentes

### Decisões duráveis
- arquitetura escolhida
- biblioteca adotada
- padrão de projeto definido
- fluxo oficial de trabalho

### Fatos persistentes
- hardware principal do usuário
- modelo principal usado no agente
- convenções do projeto

### Procedimentos comprovados
- como investigar compatibilidade
- ordem ideal de pesquisa
- fallback que costuma funcionar

## O que NÃO promover

- erros pontuais sem recorrência
- resultados temporários
- tarefas do dia já concluídas e sem valor futuro
- detalhes descartáveis
- ruído de execução
- experimentos sem conclusão

## Processo de consolidação

### 1. Ler memória operacional
Ler os logs diários mais recentes e identificar:
- decisões
- padrões
- preferências
- lições
- erros recorrentes
- próximos passos que viraram regra

### 2. Classificar cada item
Para cada item, decidir:
- episódico
- semântico
- procedural
- descartável

### 3. Avaliar durabilidade
Perguntar:
- isso será útil em sessões futuras?
- isso representa padrão estável?
- isso se aplica além do contexto de hoje?
- isso evita retrabalho?

### 4. Promover
Se for durável:
- resumir de forma curta
- escrever em MEMORY.md
- colocar na seção correta

### 5. Limpar redundância
- evitar duplicatas
- fundir itens equivalentes
- substituir versões antigas por versões melhores
